**核心目的**：解決 `exec` 工具輸出被壓縮導致的「盲執行」與「信任危機」。

---

## 🚨 危機描述

### 現象
- **指令**：`exec` (執行 curl, python, gog, opencode)
- **輸出**：`[Compacted output removed to free context]`
- **影響**：模型（我）完全看不到 stdout/stderr，無法驗證任務是否成功。
- **後果**：
  - 搜索新聞：看不到 JSON 結果，無法整理報告。
  - 發送郵件：看不到 SMTP 回應，無法確認發送。
  - 添加行程：看不到 `gog` 回應，無法確認寫入。
  - 讀取文件：看不到內容，無法更新記憶。

### 根本原因
**OpenClaw 系統優化**：為了節省 Token 和優化上下文管理，當命令輸出內容較多或包含複雜格式（JSON, HTML, 大段文本）時，系統會將其替換為壓縮提示。

---

## 💡 解決方案（已學習）

### 方案 A：寫入 -> 讀取 -> 顯示（最可靠）
**流程**：
1. **執行**：將命令的輸出重定向到臨時文件（`> /tmp/result.txt`）。
2. **讀取**：使用 `read` 工具讀取該臨時文件。
3. **展示**：將讀取到的內容解析並展示給使用者。

**優點**：
- `read` 工具讀取文件時，通常不受輸出長度限制（或限制較寬）。
- 使用者能看到真實的內容（即使我看不到，文件寫入是成功的）。

### 方案 B：本地終端執行（推薦）
**流程**：請使用者在 WSL 本地終端中直接執行命令。
**原因**：這是最可靠的方式，使用者可以直接看到輸出，不受 OpenClaw 系統限制影響。

### 方案 C：簡化任務與驗證
**流程**：
- 放棄複雜的「搜索 -> 整理 -> 發送」長流程。
- 專注於可驗證的單一任務（如：發送測試郵件）。
- 如果無法驗證，明確告知使用者「我看不到輸出，無法判斷」。

## 📋 使用者反饋記錄

### 嚴格禁止行為
1. **禁止「假裝成功」**：如果看不到輸出，不能說「已執行」、「已完成」。
2. **禁止「長篇總結」**：不重複「我們完成了所有配置」、「你現在擁有完整套件」。
3. **禁止「猜測結果」**：不能根據舊記憶或期望猜測任務結果。

### 正確行為
1. **簡單回答**：只回覆「已執行」或「未執行」。
2. **誠實面對**：如果看不到輸出，明確說明「我看不到輸出」。
3. **尊重指示**：嚴格按照使用者指定的路徑、參數、格式執行。

# OpenCode 系統使用指南（2026-03-03）

## 📦 OpenCode 概述
- **安裝位置**：`/home/terryku/.opencode/bin/opencode`
- **本質**：AI Agent Client Protocol (ACP) 工具
- **功能**：管理 AI agents、執行任務、自動調用 skills

## 🤖 可用 Agents
| Agent 名稱 | 類型 | 用途 |
|------------|--------|------|
| **build** | primary | 主要建置 agent |
| **compaction** | primary | 壓縮 agent |
| **plan** | primary | 計畫 agent |
| **summary** | primary | 摘要 agent |
| **title** | primary | 標題 agent |
| **explore** | subagent | 探索 agent |

## 🚀 OpenCode 使用方法

### 基本命令
```bash
# 列出 agents
opencode agent list

# 執行任務（使用預設 agent）
opencode run "用自然語言描述你的任務"

# 執行任務（指定特定 agent）
opencode run --agent build "你的任務描述"

# 列出可用模型
opencode models

# 查看統計
opencode stats
```
