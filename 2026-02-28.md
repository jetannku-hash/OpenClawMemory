## 學到的教訓

### OpenClaw 環境限制
1. **非標準終端**
   - OpenClaw 以 Node.js 子進程執行工具
   - 沒有標準 TTY（終端）環境
   - 導致需要 TTY 的工具無法正常運作

2. **環境變數封鎖**
   - OpenClaw 會過濾敏感環境變數
   - 像 `GOG_KEYRING_PASSWORD` 無法直接注入
   - 需要使用 systemd user-level override

### gog 工具的限制
1. **OAuth 流程複雜**
   - gog 設計為在標準終端環境使用
   - 在 WSL2 + OpenClaw 環境下需要特殊配置

2. **Keyring 密碼保護**
   - gog 使用 keyring 來存儲 OAuth Token
   - 需要密碼才能解鎖
   - 在無 TTY 環境下無法輸入密碼

### 解決方案
1. **使用 systemd override**
   - 創建 user-level systemd 配置
   - 在配置中注入環境變數

2. **使用 Windows 瀏覽器授權**
   - 在 Windows 瀏覽器中完成 OAuth 授權
   - 然後將 redirect URL 貼回給 gog

3. **文檔化配置**
   - 將所有成功的配置步驟記錄下來
   - 未來遇到問題時可以快速參考
