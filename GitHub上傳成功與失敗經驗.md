## ❌ 失敗的嘗試與原因

### 失敗案例 1：Git Push 失敗

| 嘗試方式 | 錯誤訊息 | 根本原因 |
|----------|----------|----------|
| **Git HTTPS push** | `fatal: could not read Username` | Git 需要 SSH 金鑰或使用者名稱/密碼 |
| **Git Token URL** | `fatal: could not read from remote repository` | Git 不支援 Token URL 格式 |
| **Git SSH URL** | `Host key verification failed` | SSH 金鑰未設定或未添加到 GitHub |

**根本原因：**
- ❌ Git 在非互動模式下無法正確處理認證
- ❌ 缺少 SSH 金鑰設定
- ❌ GitHub 不支援 Token URL 格式

### 失敗案例 2：GitHub API 上傳失敗

| 嘗試方式 | 錯誤訊息 | 根本原因 |
|----------|----------|----------|
| **upload_notes.sh** | `"sha" wasn't supplied (HTTP 422)` | 嘗試更新已存在的檔案，未提供 SHA |
| **upload_notes_fixed.sh** | `Could not resolve host: api.github.com` | 使用錯誤的端點 |
| **simple_upload_notes.sh** | `"sha" wasn't supplied (HTTP 422)` | 嘗試更新已存在的檔案，未提供 SHA |

**根本原因：**
- ❌ GitHub API 在更新檔案時必須提供 SHA
- ❌ 端點使用錯誤：`api.github.com` 應為 `github.com`
- ❌ 自動化腳本未正確處理已存在的檔案

## 🎯 關鍵經驗與最佳實踐

### ✅ 成功的關鍵因素

1. **使用正確的端點**
   - ✅ GitHub API：`https://api.github.com`
   - ✅ GitHub 網頁：`https://github.com`
   - ❌ 錯誤：`https://api.github.com`

2. **新檔案 vs. 已存在檔案**
   - ✅ **新檔案：** 不需要 SHA，可以直接 PUT
   - ❌ **已存在檔案：** 必須先 GET 獲取 SHA，再用 PUT 更新
   - 🔍 **關鍵：** GitHub API 對新舊檔案要求不同

3. **手動 vs. 自動化**
   - ✅ **手動上傳：** 最簡單、最可靠、最直觀
   - ❌ **自動化腳本：** 複雜、容易出錯

4. **Base64 編碼**
   - ✅ **正確：** 使用 `base64 -w 0` (無換行)
   - ❌ **錯誤：** 使用 `base64` (有換行會導致錯誤)

## 🎯 下一步最佳實踐

### 立即行動（推薦）

#### 方式 1：手動上傳（最快速、最可靠）

**操作步驟：**
1. 訪問：https://github.com/jetannku-hash/OpenClawNote
2. 點擊「Upload files」
3. 拖放：`/home/terryku/.openclaw/workspace/my_obsidian_vault/`
4. 填寫提交訊息：`初始化筆記資料庫`
5. 點擊「Commit changes」

**優點：**
- ✅ 最簡單、最直觀
- ✅ 可以看到上傳進度
- ✅ 不會遇到任何技術問題
- ✅ 支援拖放上傳整個目錄

#### 方式 2：設定 SSH 金鑰（長期最佳）

**操作步驟：**
1. 生成 SSH 金鑰
   ```bash
   ssh-keygen -t ed25519 -C "terryku@github.com"
   ```

2. 添加公鑰到 GitHub
   - 訪問：https://github.com/settings/keys
   - 點擊「New SSH key」
   - 貼上 `~/.ssh/id_ed25519.pub` 的內容
   - 點擊「Add SSH key」

3. 初始化 Git 並推送
   ```bash
   cd /home/terryku/.openclaw/workspace/my_obsidian_vault
   git init
   git add .
   git commit -m "初始化筆記倉庫"
   git remote add origin git@github.com:jetannku-hash/OpenClawNote.git
   git push -u origin master
   ```

4. 之後每次更新只需
   ```bash
   cd /home/terryku/.openclaw/workspace/my_obsidian_vault
   git add .
   git commit -m "更新筆記：$(date +'%Y-%m-%d %H:%M')"
   git push
   ```

**優點：**
- ✅ 最可靠、最快速
- ✅ 支援增量推送
- ✅ 自動處理 SHA 和衝突
- ✅ 完整的 Git 歷史記錄
